# 品川プリンスホテル 記念日フォトフレーム撮影システム
# 管理者ガイド（だれでもわかる やさしい版）

このマニュアルは、パソコンにあまり慣れていない方でも
システムの管理ができるように書かれた「やさしい版」です。

## このマニュアルに出てくる言葉の説明

はじめに、このマニュアルで使う言葉を説明します。
わからなくなったら、ここに戻って確認してください。

| 言葉 | わかりやすく言うと |
|------|-------------------|
| サーバー | インターネット上にある「データ置き場」のこと。お客様がスマホで見る画面のデータがここに入っています |
| アップロード | パソコンにあるデータを、サーバーに送ること。「ネットにデータを上げる」と同じ意味です |
| ファイル | パソコンの中のデータ1つ1つのこと。写真や文書のことです |
| フォルダ | ファイルをまとめて入れておく「引き出し」のようなもの |
| ブラウザ | インターネットを見るアプリのこと。Safari や Chrome のことです |
| URL | インターネットの住所のこと（ https://〜 で始まるもの） |
| JSON | 設定を書いてあるファイル。中身は普通の文字なのでメモ帳で開けます |
| キャッシュ | スマホが「前に見たページ」を覚えておく仕組み。これがあると更新が反映されないことがあります |
| HTTPS | 鍵マーク付きの安全なインターネット通信。カメラを使うにはこれが必要です |

## このシステムでできること

このシステムは、ホテルのレストランに来たお客様が
スマートフォンで **記念写真** を撮れるしくみです。

お客様は:
1. テーブルにある **QRコード** をスマホで読み取ります
2. **パスワード** を入力してログインします
3. スマホの **カメラ** で写真を撮ります
4. かわいい **フォトフレーム** や **フィルター** を付けられます
5. 撮った写真を **スマホに保存** できます

管理者（あなた）ができること:
- パスワードの変更
- フォトフレーム画像の入れ替え・追加
- その他の設定変更

## やりたいこと① パスワードを変える

例: HAPUNAのパスワードを「hapuna2026」から「hapuna2027」に変えたい

### 手順

**ステップ1** - 設定ファイルを開く

パソコンで以下の場所にあるファイルを開きます。

```
assets → config → restaurants.json
```

このファイルを **メモ帳**（Mac なら「テキストエディット」）で開いてください。
右クリック → 「このアプリケーションで開く」→「テキストエディット」で開けます。

**ステップ2** - パスワードを探す

ファイルの中に、こんな文字が並んでいます:

```
"id": "hapuna",
"name": "HAPUNA",
...
"password": "hapuna2026"
```

この中の `"hapuna2026"` の部分を、新しいパスワードに変えます:

```
"password": "hapuna2027"
```

**大事なルール:**
- 前後の「"」（ダブルクォーテーション）は消さないでください
- パスワードは **半角英数字** のみ使えます
- 日本語やスペース（空白）は使えません

**ステップ3** - 保存する

キーボードで:
- Mac: `Command + S`
- Windows: `Ctrl + S`

を押して保存します。

**ステップ4** - サーバーにアップロードする

変更した `restaurants.json` をサーバーにアップロードします。

**ステップ5** - キャッシュをリセットする（★重要！）

`sw.js` というファイル（一番上のフォルダにあります）を開きます。
上の方にこんな行があります:

```
const CACHE_VERSION = 'v20';
```

この数字を1つ上げます:

```
const CACHE_VERSION = 'v21';
```

この `sw.js` もサーバーにアップロードしてください。

**これをしないと、お客様のスマホに古い情報が残ったままになります！**

## やりたいこと② フォトフレームの画像を入れ替える

詳しい手順は、別冊の「フォトフレーム交換・追加マニュアル」に書いてあります。

ファイルの場所: `docs/FRAME_MANUAL_EASY.md`

ここでは **ざっくりとした流れ** だけ書きます:

1. デザイナーさんに新しいフレーム画像を作ってもらう（PNG、1080x1920、背景透明）
2. 今のフレーム画像と同じ名前にして、同じ場所に上書きする
3. サーバーにアップロードする
4. `sw.js` のバージョン番号を1つ上げて、サーバーにアップロードする

**同じ名前で上書きする場合** → 設定ファイルの変更は不要です！

---

## やりたいこと③ レストランのログインページの見え方を確認する

各レストランには、それぞれ専用のQRコードがあります。
QRコードの中身は、こういう形のインターネットアドレスです:

```
https://（サイトのアドレス）/login.html?restaurant=hapuna
```

最後の `hapuna` の部分がレストランによって変わります。
パソコンのブラウザでこのアドレスを開けば、お客様と同じ画面が見られます。

---

## やりたいこと④ QRコードを作りたい

1. パソコンのブラウザで `tools/qr-generator.html` を開きます
2. 「全レストランのQRコードを生成」ボタンを押します
3. 各レストランのQRコードが表示されます
4. 「QRコード保存」ボタンで画像として保存できます
5. 「印刷用ページを開く」で全レストラン分を一気に印刷できます

## 困ったときの確認リスト

| 困りごと | まず確認すること |
|---------|-----------------|
| お客様がログインできない | パスワードが合っているか `restaurants.json` を確認。全角/半角の入力ミスが多い |
| 画面が変わらない・古いまま | `sw.js` のバージョン番号を上げてサーバーにアップロードしたか |
| 画面が真っ白になった | JSONファイルの書き方に間違いがないか https://jsonlint.com/ で確認 |
| フレームが表示されない | フレーム画像がサーバーの正しい場所に入っているか確認 |
| カメラが動かない | サイトがHTTPS（鍵マーク付き）でアクセスされているか確認 |

上の方法で解決しない場合は、システム開発担当者にご連絡ください。

---

## ★ 最も大事なルール（これだけ覚えてください）

```
何かを変更したら → sw.js のバージョン番号を1つ上げる → サーバーにアップロードする
```

これを忘れると、お客様のスマホに変更が反映されません。
逆に言えば、これさえ忘れなければ大丈夫です。
