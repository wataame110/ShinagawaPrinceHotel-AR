# フォトフレームの交換・追加マニュアル

## はじめに読んでください

このマニュアルは「フォトフレームの画像を入れ替えたい」「新しいフレームを増やしたい」ときに読む説明書です。

パソコンやプログラミングの知識は一切必要ありません。
「ファイルの名前を変える」「文字を少し書き換える」だけでできます。

---

## よく使う言葉のかんたん説明

| この説明書で出てくる言葉 | わかりやすく言うと |
|---|---|
| フォルダ | 書類を入れる棚の引き出しのようなもの。ファイルを整理するための入れ物です |
| ファイル | 写真や書類などのデータ1つ1つのこと |
| PNG（ピーエヌジー） | 写真のデータの種類の名前。背景を透明にできる特別な写真の形式です |
| JSON（ジェイソン） | 設定が書いてあるメモ帳のようなファイル。中身は普通の文字です |
| パス | ファイルがどの引き出しのどこにあるかを示す住所のようなもの |
| サーバー | インターネット上にデータを置いておく場所。ホームページが動いている場所 |

---

## このシステムのフレーム画像の置き場所

パソコン上のファイル一覧を見ると、こんな構造になっています。

```
[メインフォルダ]
  └── assets
        └── images
              └── frames
                    ├── common          ← ★ 全店共通のフレーム画像はここ
                    └── restaurants     ← ★ 各レストラン専用のフレーム画像はここ
                          ├── hapuna        ← HAPUNA専用
                          ├── table9        ← TABLE 9専用
                          ├── jojoen        ← 叙々苑専用
                          └── ...（他のレストランも同様）
```

そして「設定ファイル」が1つあります。

```
[メインフォルダ]
  └── assets
        └── config
              └── frames-config.json   ← ★ この1つのファイルだけ触ります
```

# やりたいこと別・手順書

---

## やりたいこと (1) 共通フレームの画像を別の画像に交換したい

例：「ピンクフローラル」というフレームの画像を、新しいデザインに変えたい場合

### 手順

#### STEP 1: 新しい画像を用意する

フレーム画像には決まりがあります。

```
  ・形式 ........................ PNG（ピーエヌジー）
  ・写真のたて横の大きさ ......... よこ 1080 ドット × たて 1920 ドット
  ・背景 ........................ 透明（写真が写る部分は透明になっている画像）
```

デザイナーさんに「よこ1080、たて1920のPNGで、背景は透明にしてください」と伝えれば大丈夫です。


#### STEP 2: ファイルの名前を今と同じ名前にする

今使っている画像と全く同じファイル名にしてください。

例えば「ピンクフローラル」は「common_iy02.png」という名前です。
新しい画像のファイル名も「common_iy02.png」にしてください。

>>> どのフレームがどのファイル名か？の一覧表はこのマニュアルの最後にあります <<<


#### STEP 3: ファイルを上書きする

パソコンのフォルダで以下の場所を開きます。

```
assets → images → frames → common
```

この中に、同じ名前のファイルが既にあるので、新しいファイルをここに入れます。
「上書きしますか？」と聞かれたら「はい」を選んでください。


#### STEP 4: サーバーにアップロードする

変えたファイルだけをサーバーにアップロードすれば完了です。

>>> ファイル名を同じにして上書きした場合は、これだけで終わりです <<<
>>> 設定ファイルの変更は必要ありません <<<

## やりたいこと (2) 共通フレームを新しく増やしたい

例：新しいデザインのフレームを追加して、全部で12種類にしたい場合

### 手順

#### STEP 1: 新しい画像を用意する

（やりたいこと(1)のSTEP 1と同じ。PNG、1080x1920、背景透明）


#### STEP 2: ファイル名を決める

ルールは簡単です。

```
common_○○○.png
```

「common_」で始めて、その後ろに好きな名前を英語で付けます。
日本語は使えません。スペース（空白）も使えません。

良い例:
  common_sakura.png
  common_birthday_gold.png
  common_summer2026.png

ダメな例:
  桜フレーム.png          ← 日本語はダメ
  common sakura.png       ← スペースはダメ
  sakura.png              ← 「common_」が付いてない


#### STEP 3: 画像をフォルダに入れる

パソコンのフォルダで以下の場所を開いて、ファイルを入れます。

```
assets → images → frames → common
```


#### STEP 4: 設定ファイルに1行追加する（ここが一番大事！）

「frames-config.json」という設定ファイルを開きます。
場所は:

```
assets → config → frames-config.json
```

このファイルをメモ帳（テキストエディタ）で開くと、
こんな感じの文字がたくさん並んでいます。

```
  "commonFrames": [
    {
      "id": "common_iy02",
      "name": "ピンクフローラル",
      "path": "assets/images/frames/common/common_iy02.png",
      "thumbnail": "assets/images/frames/common/common_iy02.png",
      "description": "淡いピンクの花々が彩る水彩風フレーム"
    },
    ここに他のフレームの情報がたくさん続きます...
  ],
```

一番最後のフレーム（閉じカッコ } の後ろ）にカンマ「,」を付けて、
以下の形で新しいフレームの情報を追加してください。

```
    ,
    {
      "id": "ここにファイル名から.pngを取った名前",
      "name": "ここに画面に表示したい名前（日本語OK）",
      "path": "assets/images/frames/common/ここにファイル名.png",
      "thumbnail": "assets/images/frames/common/ここにファイル名.png",
      "description": "ここに説明文（日本語OK）"
    }
```

具体例（common_sakura.pngを追加する場合）:

```
    ,
    {
      "id": "common_sakura",
      "name": "さくらフレーム",
      "path": "assets/images/frames/common/common_sakura.png",
      "thumbnail": "assets/images/frames/common/common_sakura.png",
      "description": "桜が舞い散る春らしいフレーム"
    }
```

---- 書くときの注意点 ----

  ★ 「"」（ダブルクォーテーション）は絶対に消さないでください
  ★ 最後の行の「}」の後ろにカンマ「,」を付けないでください
  ★ 全角の「"」ではなく、半角の「"」を使ってください
  ★ 1文字でも間違えると壊れることがあります。不安なら詳しい人に頼んでください。

---- 壊れたかどうか確認する方法 ----

https://jsonlint.com/ にアクセスして、
frames-config.json の中身を全部コピーして貼り付けて
「Validate JSON」ボタンを押してください。
「Valid JSON」と出たらOK。エラーが出たらどこか間違えています。


#### STEP 5: サーバーにアップロードする

以下の2つのファイルをサーバーにアップロードします。

```
  1. 新しい画像ファイル（common_sakura.png）
  2. 変更した設定ファイル（frames-config.json）
```


#### STEP 6: キャッシュをリセットする（重要！）

「sw.js」というファイルを開いてください（メインフォルダの直下にあります）。

ファイルの上の方に、こんな行があります。

```
const CACHE_VERSION = 'v20';
```

この数字を1つ上げてください。

```
const CACHE_VERSION = 'v21';
```

この「sw.js」もサーバーにアップロードしてください。
これをしないと、お客様のスマホに古い情報が残ったままになります。

## やりたいこと (3) レストラン専用フレームを設定したい

例：HAPUNAのオリジナルフレームを設定する場合

### 手順

#### STEP 1: 画像を用意する（共通フレームと同じルール）

PNG、1080x1920、背景透明。


#### STEP 2: レストラン専用フォルダを用意する

```
assets → images → frames → restaurants → hapuna
```

「hapuna」というフォルダがなければ、新しく作ってください。


#### STEP 3: 画像ファイルを入れる

好きなファイル名でOK（英語のみ）。例:

```
hapuna_frame1.png
hapuna_frame2.png
```


#### STEP 4: 設定ファイルを書き換える

「frames-config.json」を開くと、各レストランの情報が書かれています。

hapunaの部分を探してください:

```
  "hapuna": [
    {
      "id": "hapuna_original_1",
      "name": "HAPUNA オリジナル①",
      "path": "assets/images/frames/restaurants/placeholder_1.png",    ← ここを変える
      "thumbnail": "assets/images/frames/restaurants/placeholder_1.png" ← ここを変える
    },
```

「placeholder_1.png」の部分を、実際のファイルの場所に変えます:

```
      "path": "assets/images/frames/restaurants/hapuna/hapuna_frame1.png",
      "thumbnail": "assets/images/frames/restaurants/hapuna/hapuna_frame1.png"
```

★ 変えるのは「path」と「thumbnail」の2箇所だけです


#### STEP 5: サーバーにアップロードする

以下をアップロード:
  1. 画像ファイル
  2. frames-config.json
  3. sw.js（キャッシュバージョンを1つ上げてから）

---

## やりたいこと (4) フレームを減らしたい（使わなくしたい）

設定ファイルからそのフレームの情報を丸ごと消せばOKです。
画像ファイルは消しても消さなくても、どちらでも構いません。

ただし、消す場所を間違えると壊れることがあるので、
不安な場合は詳しい人にお願いしてください。

# 困ったときの確認ポイント

| 困りごと | 確認すること |
|---|---|
| 新しいフレームが表示されない | 画像ファイルはちゃんとサーバーにある？ |
| | frames-config.json のパスに打ち間違いはない？ |
| | sw.js のバージョン番号を上げた？ |
| 画像が変わらない | sw.js のバージョン番号を上げてアップロードした？ |
| 画面が真っ白になった | frames-config.json に書き間違いがある可能性大。jsonlint.com で確認 |
| フレームの枠が黒くなる | 画像の背景が透明になっていない。デザイナーに確認 |
| 画像が荒い・ぼやける | 画像サイズが小さすぎる。よこ1080 x たて1920以上で作り直し |

# まとめ: やること早見表

| やりたいこと | やること |
|---|---|
| 画像を入れ替える（同じ名前で上書き） | 画像を上書き → サーバーにアップ → sw.jsの番号を上げる |
| 画像を入れ替える（別の名前で） | 画像を配置 → frames-config.jsonのパスを書き換え → サーバーにアップ → sw.jsの番号を上げる |
| 新しいフレームを追加する | 画像を配置 → frames-config.jsonに情報を追加 → サーバーにアップ → sw.jsの番号を上げる |
| フレームを減らす | frames-config.jsonから情報を削除 → サーバーにアップ → sw.jsの番号を上げる |

★ どの作業でも、最後に「sw.jsの番号を1つ上げてアップロード」を忘れずに！
